### 技術架構規劃重點

| 層級 | 內容 |
|------|------|
| Domain | `AuctionItem`（Active/Ended 狀態機）、`AuctionImage`、`Category`、`Watchlist`、`AuctionWonEvent` |
| Application | Commands（CreateAuction, UpdateAuction）、Queries（GetAuctions, GetAuction, GetWatchlist）、Validators、`IBiddingQueryService` stub |
| Infrastructure | `AuctionDbContext`（schema: auction）、Migrations（索引設計）、`AuctionEndBackgroundService`（PeriodicTimer 60s）|
| API | `AuctionsController`、`WatchlistController`、Response DTOs |
| Tests | Unit（狀態機、Handlers、Validators）+ Integration（Testcontainers，全 US 覆蓋） |

### API 端點規劃（依競標者優先順序）

| 方法 | 端點 | 說明 |
|------|------|------|
| GET | `/api/auctions` | 商品列表（q/category/page/pageSize） |
| GET | `/api/auctions/{id}` | 商品詳情（含最高出價） |
| POST | `/api/auctions/{id}/watchlist` | 加入追蹤清單 |
| DELETE | `/api/auctions/{id}/watchlist` | 移除追蹤清單 |
| GET | `/api/watchlist` | 查詢追蹤清單（?status=active） |
| POST | `/api/auctions` | 建立商品（直接 Active） |
| PUT | `/api/auctions/{id}` | 編輯商品（限非競標敏感欄位） |

### 狀態機（最終確認）
```
[建立] → Active → [結標時間到/排程] → Ended
```
（無 Draft，無 Cancelled，無逆轉）
