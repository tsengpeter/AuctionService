## 今日完成事項

### 1. Member 模組規格完善（spec.md）

完成多輪 `/speckit.clarify` 澄清，spec.md 最終確立以下內容：

| 事項 | 決策 |
|------|------|
| 必填欄位 | 註冊需填 email、username、password、phoneCountryCodeId、phoneNumber、address 物件（共 6 項） |
| 地址儲存 | 結構化子欄位：`address_country`、`address_city`、`address_postal_code`、`address_line`（各子欄均選填） |
| 電話號碼格式 | 僅接受純數字；09 開頭自動去除前導 `0` 後儲存（`0912345678` → `912345678`） |
| 電話國碼 | 獨立 `phone_country_codes` 查詢表（id, dialCode, countryName, countryIso），users 儲存 FK id |
| 密碼變更 | 本期納入（US7）；成功後撤銷該使用者所有 Refresh Token，強制所有裝置重新登入 |
| username 唯一性 | 大小寫不敏感（`username_normalized` 欄位存小寫） |
| username 長度 | 3–30 字元 |
| Token Rotation | 每次 refresh 撤銷舊 token；已撤銷 token 重用直接回 401，不做 mass revocation |
| IP 速率限制 | 同 IP 每分鐘登入失敗 > 5 次 → 429，不鎖帳號 |
| Token 清理 | IHostedService 每日背景清理過期 + 已撤銷的 Refresh Token |

**最終 User Stories（7 個）**：US1 註冊、US2 登入、US3 Refresh Token、US4 登出、US5 查詢個人資料、US6 更新個人資料、US7 變更密碼

**最終 Functional Requirements（16 條）**：FR-001 ~ FR-016（FR-016 為國碼查詢端點）

## 技術架構確認

```
member.phone_country_codes  ←── (FK) ── member.users ──── (1:N) ──→ member.refresh_tokens
  id (INT, PK, identity)              id (UUID, PK)                  id (UUID, PK)
  dial_code (VARCHAR 10, UNIQUE)      email (UNIQUE)                 user_id (logical FK)
  country_name (VARCHAR 100)          username_normalized (UNIQUE)   token_hash (UNIQUE)
  country_iso (CHAR 2)                phone_country_code_id (INT FK) expires_at
                                      phone_number (VARCHAR 30)      is_revoked
                                      address_* (4 nullable cols)
```
