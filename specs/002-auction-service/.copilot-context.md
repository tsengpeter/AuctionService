## 📋 核心技術棧

### 語言與框架
- **C# 12** / **.NET 9**
- **ASP.NET Core 9 Web API** (Controller-based，**不使用 Minimal APIs**)
- **EF Core 9.0** (Code-First workflow)

### 資料庫與快取
- **PostgreSQL 16** - 主資料庫（Snowflake ID 主鍵、全文搜尋 tsvector + GIN Index）
- **Redis 7** - 分散式快取（Bidding Service 降級機制，3秒逾時 + 5分鐘 TTL）

### 儲存與外部服務
- **AWS S3 / Azure Blob Storage / MinIO** - 拍賣圖片儲存（最多 5 張，5MB 限制）
- **Bidding Service** - HTTP 整合（Polly 重試策略，快取降級）

### 測試框架
- **xUnit 2.x** - 測試框架
- **Moq 4.x** - Mock 框架
- **FluentAssertions 6.x** - 斷言庫
- **Testcontainers** - PostgreSQL + Redis 整合測試

### 關鍵套件
- **IdGen 3.x** - Snowflake ID 生成器（64-bit Long）
- **FluentValidation 11.x** - 請求驗證
- **Polly 8.x** - 彈性與重試策略
- **Serilog 4.x** - 結構化日誌

## 📊 資料模型

### 核心實體 (4 個)

1. **Auction (拍賣商品)** - 主要聚合根
   - `Id` (long, Snowflake ID)
   - `Title`, `Description`, `StartingPrice`, `CategoryId` (FK)
   - `StartTime`, `EndTime` (TIMESTAMPTZ)
   - `CreatorId` (FK to Member Service User)
   - `ImageUrls` (JSONB array, max 5)
   - `SearchVector` (tsvector, 全文搜尋)
   - `RowVersion` (byte[], 樂觀鎖定)
   - `Status` (computed property: Pending/Active/Ended)

2. **Category (分類)** - 預先定義
   - `Id` (long, Snowflake ID)
   - `Name` (unique), `DisplayOrder`, `IsEnabled`
   - 6 個預設分類：電子產品、家具、收藏品、服飾、書籍、其他

3. **Follow (追蹤)** - 使用者與拍賣關聯
   - `Id` (long, Snowflake ID)
   - `UserId`, `AuctionId` (FK)
   - Unique constraint: (UserId + AuctionId)

4. **ResponseCode (回應代碼)** - 統一 API 回應管理
   - `Code` (PK string)
   - `MessageZhTw`, `MessageEn` (多語言支援)
   - `Category`, `Severity`
   - 10 個預設代碼 (SUCCESS, AUCTION_NOT_FOUND, 等)

## 🔌 API 端點 (12 個)

### Auctions (8 endpoints)
- `GET /api/auctions` - 列表（分頁、篩選、搜尋、排序）
- `POST /api/auctions` - 建立（需驗證）
- `GET /api/auctions/{id}` - 詳細資訊
- `PUT /api/auctions/{id}` - 更新（需驗證、擁有者限定、樂觀鎖定）
- `DELETE /api/auctions/{id}` - 刪除（需驗證、擁有者限定）
- `GET /api/auctions/{id}/current-bid` - 取得當前出價（快取降級支援）
- `POST /api/auctions/{id}/images` - 上傳圖片（需驗證、multipart/form-data）
- `GET /api/auctions/users/{userId}` - 使用者的拍賣商品

### Categories (1 endpoint)
- `GET /api/categories` - 列表

### Follows (3 endpoints)
- `GET /api/follows` - 追蹤清單（需驗證）
- `POST /api/follows` - 追蹤商品（需驗證）
- `DELETE /api/follows/{auctionId}` - 取消追蹤（需驗證）

### Health (2 endpoints)
- `GET /health` - 健康檢查
- `GET /ready` - 就緒檢查（含資料庫連線驗證）

## 📚 關鍵技術決策 (10 個)

1. **Snowflake ID 主鍵策略** - IdGen 套件、64-bit Long、時間排序
2. **樂觀鎖定並行控制** - EF Core RowVersion、低衝突場景
3. **Redis 快取降級** - Bidding Service 不可用時從快取讀取、3秒逾時
4. **PostgreSQL 全文搜尋** - tsvector + GIN Index、保留 Elasticsearch 遷移路徑
5. **雲端物件儲存** - S3/Blob/MinIO、資料庫僅存 URL
6. **計算式商品狀態** - 查詢時動態計算、無背景任務、100% 準確
7. **ResponseCodes 表** - 集中管理、多語言支援
8. **Clean Architecture** - 4 層結構、依賴反轉
9. **手動 POCO 映射** - 無 AutoMapper、效能優先
10. **Controller-Based API** - 非 Minimal APIs、結構化設計

### 撰寫程式碼時
- 優先使用 **Clean Architecture** 分層結構
